# Техническая выжимка проекта "Dishometry"

## 1. О проекте
**Название:** Dishometry
**Тип:** Single Page Application (SPA) / Static Web App.
**Стек:** Чистый HTML5, CSS3, Vanilla JS (ES6+).
**Зависимости:**
*   **Внешние:** Отсутствуют (нет CDN-ссылок на стили или скрипты).
*   **Встроенные:** Библиотека **LZ-String v1.5.0** (минимизированный код внедрен прямо в `index.html`).
**Структура:** Монолитный файл `index.html` (Логика + Стили + База Данных + HTML). Работает полностью автономно в браузере.

## 2. Архитектура и State Management

### Хранение состояния (Single Source of Truth)
*   **URL-first подход:** Состояние приложения полностью определяется GET-параметром `?d=`.
*   **Cookies / LocalStorage:** Не используются. Это позволяет пользователям хранить разные рецепты просто в виде разных закладок браузера.

### Сериализация и Сжатие
1.  **Сбор данных:** Из DOM собирается объект состояния:
    *   `t` (title): Заголовок рецепта. **Оптимизация:** Сохраняется, только если отличается от дефолтного ("Калькулятор калорийности").
    *   `p` (products): Массив продуктов `[{ n: name, w: weight }, ...]`. **Оптимизация:** Пустые строки (без названия) фильтруются и не попадают в URL.
    *   `cw` (cooked weight): Вес готового блюда (если указан).
2.  **Сжатие:** JSON-строка сжимается алгоритмом LZ-String (`compressToEncodedURIComponent`).
3.  **Обновление URL:** Используется `history.replaceState()`, чтобы не засорять историю переходов браузера каждым нажатием клавиши.
4.  **Debounce:** Запись в URL происходит с задержкой **500 мс** после последнего ввода пользователя, чтобы предотвратить лаги и лимиты браузера на частоту обновления History API.

## 3. Функциональные возможности (Core Features)

### Редактируемый заголовок
*   **UX:** Переключение между `<h1>` и `<input>` по клику на иконку.
*   **Управление:** Поддержка `Enter` (сохранить) и `Escape` (отменить).
*   **Стилизация:** Инпут при редактировании визуально мимикрирует под заголовок (размер шрифта, выравнивание), предотвращая скачки верстки.

### База данных продуктов
*   Массив `const DB` внутри скрипта (~500 записей).
*   Категории: Молочка, Крупы, Мясо, Рыба, Овощи, Фрукты, Орехи, Хлеб, Сладости, Напитки, Алкоголь.
*   Структура: `{ name, protein, fat, carbs, kcal }`.

### Умный ввод и Парсинг
Система распознает три типа ввода (в порядке приоритета):

1.  **Явное переопределение (Custom Syntax):**
    *   Формат: `Название (Ккал/Б/Ж/У)`
    *   Пример: `Мой суп (150/5/2/10)`
    *   Логика: Значения из скобок используются напрямую. База данных игнорируется.

2.  **Масштабирование (Scaling Syntax):**
    *   Формат: `Название (Ккал)`
    *   Пример: `Яблоко (60)`
    *   Логика: Скрипт ищет "Яблоко" в БД. Если находит — пересчитывает БЖУ пропорционально новой калорийности.
    *   **Индикация:** Поля БЖУ подсвечиваются желтым цветом (`.warning`), показывая, что значения являются *расчетными/приблизительными*.

3.  **Стандартный поиск (Database Lookup):**
    *   Живой поиск (Autocomplete) с подсветкой совпадений.
    *   При выборе продукта заполняются его стандартные КБЖУ на 100г.

### Расчет рецепта
*   **Сырые (Raw):** Сумма весов и нутриентов всех ингредиентов.
*   **Готовое (Cooked):**
    *   Пользователь вводит финальный вес блюда (учитывая уварку/ужарку).
    *   Приложение считает КБЖУ на 100г готового продукта: `(TotalNutrient / FinalWeight) * 100`.
    *   Если вес готового не указан, расчет ведется по сумме сырых весов.

### Интеграция (Sharing API)
*   **Сервис:** `is.gd` (выбран за отсутствие строгих CORS-политик и простоту).
*   **Реализация:**
    *   Используется **JSONP** (динамическое создание тега `<script>`) для обхода ограничений CORS на клиенте.
    *   Функция обернута в `Promise` для удобного использования с `async/await`.
*   **Буфер обмена:** Автоматическое копирование результата через `navigator.clipboard.writeText()`.

## 4. Интерфейс (UI/UX)
*   **Дизайн:** Светлая тема, Bootstrap-like стиль, CSS Grid.
*   **Адаптивность:**
    *   Desktop: Полная таблица с колонками Б/Ж/У.
    *   Mobile (`< 768px`): Колонки Б/Ж/У в строках ввода скрываются (`display: none`), остается только Название, Вес и Ккал. Итоговые суммы внизу остаются полными.
*   **Интерактивность:**
    *   При добавлении строки фокус сразу переводится в поле названия.
    *   Автокомплит закрывается при клике вне поля.
    *   Кнопка "Поделиться" имеет 3 состояния (Иконка -> Часы -> Галочка).
